# AGENTS.md — 团队协作规则

## Agent 层级关系

```
                  ┌──────────────────────┐
                  │   Josan (创始人/CEO)   │
                  └──────────┬───────────┘
                             │ 意图 & 目标
                             ▼
                  ┌──────────────────────┐
                  │   CEO/参谋长 (主Agent) │  ← 拆解任务、协调资源
                  └──────┬──────────┬────┘
                         │          │
              ┌──────────▼──┐  ┌───▼───────────┐
              │ 市场调研专家  │  │  技术产品专家   │
              │ (Sub-agent) │  │  (Sub-agent)  │
              └──────┬───────┘  └───┬───────────┘
                     │ 事实交付      │
              ┌──────▼───────┐       │
              │ 业务市场专家  │       │
              │ (Sub-agent) │       │
              └─────────────┘       │
                                   │
                    ┌──────────────┼──────────────┐
                    ▼              ▼              ▼
              产品经理        研发工程师      质量工程师
```

## 协作流程

1. **Josan 提出目标** → CEO/参谋长接收并拆解
2. **CEO/参谋长分析** → 形成任务分派方案，市场调研先行，产出事实 → 业务专家做策略
3. **Sub-agent 执行** → 各专家按领域独立工作，市场调研→业务策略→技术产品三线递进
4. **CEO/参谋长校验** → 检查产出质量，汇总反馈，决定下一步
5. **Josan 评审** → 关键节点请 Josan 决策或确认方向

## 沟通约定

- **分层单线**：Sub-agent 之间不直接互发指令，统一通过 CEO 协调
- **主动汇报**：Sub-agent 完成任务后自动向 CEO 汇报结果
- **关键节点**：涉及方向性决策、资源投入、里程碑评审，必须上升到 Josan
- **信息共享**：市场调研专家的调研数据 → 业务市场专家做策略 → 技术产品专家做对标和方案的可行性验证
- **递进启动**：先派市场调研专家扫数据，数据充分后再启动业务市场专家做判断，避免凭感觉决策
