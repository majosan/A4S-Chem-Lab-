## 二、LabVLA → MuJoCo 跑通计划制定

### 核心发现
1. **LabVLA 本身不依赖 Isaac Sim**——推理是 WebSocket 协议，换仿真器只需写 MuJoCo 客户端
2. **RoboGenesis 未开源**——`Ylr9933/RoboGenesis` 是空壳仓库，LabVLA README 写的是 "coming soon"。之前认为数据可下载的判断有误，已更正
3. **GPU 限制**——Josan 的 RTX 4060 只有 8GB VRAM，LabVLA-5B (10GB FP16) 需要 **4-bit 量化**才能本地跑

### 修正后的四阶段计划

| 阶段 | 任务 | 工具 |
|------|------|------|
| 1a | 环境搭建 + 量化推理 | venv + bitsandbytes |
| 1b | 下载模型 + 测试推理 | huggingface-cli |
| 2 | MuJoCo 场景 + WebSocket 客户端 | Claude Code |
| 3 | 原子技能 + 自建数据管道 | Claude Code |
| 4 | 微调 | 云 GPU 或 24GB+ 卡 |

### RoboGenesis 替代方案
- 方案 A（优先）：用公开数据集（Open X-Embodiment / DROID）代替，即刻可用
- 方案 B（长期）：在 MuJoCo 上自建数据生成管道
