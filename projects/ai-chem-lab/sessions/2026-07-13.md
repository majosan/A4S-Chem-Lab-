## 二、12×12 触觉网格编码方案 ✅ 定稿

### 决策结果
**确定走 12×12 网格路线**（124 个压力点映射到 144 格，其余 20 格填 0）

### 编码方案选择
**走方案 B（3-channel）**：

| 通道 | 内容 | 源码 | 物理含义 |
|------|------|------|---------|
| Ch1 | P(t) | ADC 当前值 | 瞬时接触强度 (0-255) |
| Ch2 | ΔP/Δt | P(t)-P(t-1) | 压力变化率 |
| Ch3 | σ²(P) | var(P, t-4~t) | 局部振动/纹理 |

### CNN 架构
```
Input: 12×12×3
 → Conv2D 3×3, in=3→out=16, ReLU, MaxPool 2×2 (→6×6×16)
 → Conv2D 3×3, in=16→out=32, ReLU, MaxPool 2×2 (→3×3×32)
 → Flatten → 288 → Linear → 64 (最终特征向量)
```

### 映射布局设计
手型拓扑排列：5 指纵向分布 + 掌心中央集中 + 四角 0 填充
- 行 0-2：食/中/无/小指尖端
- 行 3-4：拇指
- 行 5：指根过渡
- 行 6-9：掌心
- 行 10-11：掌心下区 + 填充

### 基础特征
- 64 维触觉特征 → 与 1024 维视觉特征拼接 → 送入 DiT
- 触觉:视觉 ≈ 1:16 比例

## 五、编码融合深度论证（已获得 Josan 理解 ✅）

### 核心论点
> 必须编码+融合，因为 LabVLA 模型输入接口是固定的（3 camera + 1 state + 1 text），没有给触觉数据留入口。
> 编码（CNN 压缩 12×12→64 维）是造入口；融合（特征拼接 1024+64+8）是让入口进来的信息和视觉对齐。
> 不做这一步，触觉数据来了模型也看不懂。

### 三个不能独立发送的原因
1. **模型架构写死**：state_dim=8 是编译时常数，改不了
2. **不改架构只能挤进 state**：144 个有空间结构的压力值塞进 8 维 MLP → 丢失空间关系
3. **改了架构要重训练**：退回到重新预训练，不是 Phase 4 微调能解决的
