## 一、项目背景

### 我们是谁

「AI 化学实验室」—— 打造高通量无人化学实验平台。联合浙江大学，聚焦**催化剂合成**的高通量自动化实验。

### 我们要干什么

```
研究员操作（触觉手套+IMU 示教） → 触觉+视觉+IMU 多模态数据 →
编码融合 → 喂入 VLA 模型微调 → 机械臂自动执行
```

- **物理实验室已就绪**（浙大）：表征设备 + 实验平台已搭建
- **缺口**：具身智能/机器人自动化层 —— 当前正在填补

### 核心技术路线

| 层级 | 选型 |
|------|------|
| 模型框架 | **LabVLA**（浙大+上海 AI Lab 开源） |
| 视觉骨干 | Qwen3-VL-4B |
| 动作专家 | DiT + Flow Matching |
| 仿真引擎 | MuJoCo |
| 数据引擎 | RoboGenesis / 自建管道 |
| 触觉数据 | 124 点柔性织物手套（VQ-VAE+CNN 编码） |

### 为什么选 LabVLA

- **全开源**，代码、模型权重均可用
- **专为科学实验室场景设计**（浙大出品，方向天然契合）
- 三阶段训练（预训练→后训练→微调）支持仿真数据→真实数据迁移
- 有 Franka 真机验证经验

### 长远目标

在 LabVLA 框架基础上，接入**触觉手套多模态数据**（124 点触觉 + IMU + 视觉），形成「人机协同的催化剂高通量合成平台」。

## 二、Phase 1 任务目标

### 为什么先做 Phase 1

**在投入硬件采购和大量研发资源之前，先用最小成本回答几个关键问题：**

1. ❓ LabVLA-5B（50 亿参数）量化后能否在 RTX 4060 (8GB) 上顺畅推理？
2. ❓ 它的 WebSocket 通信接口格式是否适合后续接入多模态数据（触觉/IMU）？
3. ❓ 模型输出的 action 结构（维度、chunk_size、数值范围）可否对齐机械臂控制需求？
4. ❓ 从「观测输入 → 模型推理 → 动作输出」的完整闭环延迟是否可接受？

**Phase 1 通过 = ✅ 框架可行，可以放心投入 Phase 2+**  
**Phase 1 不通过 = ❌ 及时调头，避免资源浪费**

## 三、任务清单

### Step 1：环境验证

**目的**：确认基础开发环境完整可用。

**要求**：
- 确认 `.venv` 中 Python 3.10 可用
- 确认 PyTorch + CUDA 正常，GPU 可调用
- 确认所有关键依赖（transformers、bitsandbytes、flash_attn、msgpack、websockets 等）已安装
- 遇到缺漏自行补装

**验收标准**：
- ✅ CUDA available = True
- ✅ GPU 识别为 RTX 4060，VRAM 8GB
- ✅ Python 3.10

---

### Step 2：下载模型 + 量化推理

**目的**：验证 LabVLA-5B-Base 能在 8GB 显存限制下完成推理。

**要求**：
- 从 HuggingFace 下载 `zjunlp/LabVLA-5B-Base`（约 9.5GB）
- 以 4-bit 量化配置加载模型（BitsAndBytes）
- 用随机图片 + 文本指令跑一次推理
- 报告：模型加载时间、推理耗时、**峰值显存占用**、输出 token 序列

**验收标准**：
- ✅ 峰值显存 < 7.5 GB（留余量）
- ✅ 推理输出合理（有连续 token，非乱码）
- ✅ 能大致理解输出格式（text 输出？action token？还是两者都有？）

---

### Step 3：WebSocket 推理闭环

**目的**：打通「模拟观测 → 模型推理 → 动作输出」的完整通信链路，确认接口兼容性。

**要求**：
- 部署 LabVLA 推理服务（WebSocket，监听 8000 端口）
- 编写测试客户端模拟发送观测数据（RGB 图像 + 文本指令）
- 接收模型返回的 action，解析并记录格式

**重点记录**：
- action 的 **shape**（几维？chunk_size 多少？）
- action 的 **数据类型**（float32？float16？）
- action 的 **数值范围**
- 一次往返的 **延迟**

**验收标准**：
- ✅ 至少完成 1 次完整观测 → 动作回复往返
- ✅ action 格式可解析，shape/dtype/range 明确
- ✅ 服务稳定运行不 OOM
