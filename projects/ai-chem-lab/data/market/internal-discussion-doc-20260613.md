#### Chemspeed 的定位（最直接的参照）

Chemspeed 是行业最成熟的玩家（25年），它解决什么问题：
> "已知配方的标准化批量实验"

它的局限：
- 只能用标准夹爪/气动工具（不是灵巧手），只能处理标准器皿
- 需要工程师编程配置新实验流程
- 无法捕捉或利用触觉信息
- 无法从人类演示中学习
- 不能自适应不同配方的物理特性差异

### 2.2 欧世盛7阶段模型：行业的技术天花板

欧世盛将高通量实验室发展划分为7个阶段，对我们非常有参考价值：

| 阶段 | 名称 | 描述 | 谁到了？ |
|:----:|:-----|:-----|:--------:|
| 1-2 | 基础自动化 | 机械替人、单步自控 | 大多数企业 |
| 3 | 单步反应自动化 | 微通道反应器层面 | 欧世盛 |
| 4 | 部分反应全流程 | 全流程打通（黑灯实验室） | 欧世盛（少数商业案例） |
| **5** | **全反应类型覆盖** | **超低温、超高压、聚合等** | **"全球无人突破"** |
| 6-7 | AI融合→全自主 | 干湿实验室融合 | 未来方向 |

**意义：** 行业做到阶段4就已经卡住了，阶段5全世界突破不了。**不是在现有路线的基础上"更自动化"就能解决问题**——需要新路线。我们的人机协作路线提供了"绕开阶段5瓶颈"的可能性。

### 2.3 触觉HRI / 灵巧手在哪些领域验证了价值？

我们研究了三个成功案例，总结出共同规律：

#### 案例A：手术机器人 Da Vinci
- **形态**：医生远程操控+触觉反馈
- **成功原因**：不是"模拟人手"，而是"给了医生超能力"（震颤过滤、运动缩放、微创）
- **核心公式**：纯人做不好（有震颤） + 纯自动化解不了（手术需要人类判断） → 人机协作

#### 案例B：工业力控装配（ABB/FANUC）
- **形态**：协作臂+腕部力传感器，自适应调整
- **成功原因**：零件公差导致纯刚性自动化会失败，触觉让机器人能"感觉"到配合状态
- **核心公式**：纯自动化解不了（零件有公差） + 纯人手效率低 → 人机协作

#### 案例C：核废料/炸弹拆解遥操作
- **形态**：人遥控+力反馈到操作手柄
- **成功原因**：人不能靠近，但操作太复杂不能全自动
- **核心公式**：纯人不能做（辐射环境） + 纯自动化解不了（复杂操作） → 人机协作

#### 提炼出的判断框架

```
在某个环节：
  纯自动化解不了  AND  纯人做不好/做不了
     → 人机协作创造价值
```

### 2.7 关于人形机器人是否该用于实验室的结论

调研显示人形机器人（Figure、Tesla Optimus等）当前的实际价值场景是：
- **制造业工厂**（搬运、装配、上下料）
- **物流仓库**（搬运、分拣）
- **危险环境**（探索、巡检）

**实验室场景 <— 目前没有人形机器人进入这个领域。**

原因是实验室需要的不是"会走路的机器人"，而是"能精细操作的机械系统"。人的作用不是在实验台之间走动（AGV可以），而是在实验操作中做判断和精细调整。

**对你们的启示：不要做人形机器人，做实验室专用的人机协作系统。** 这是更务实的路线。

### 2.8 行业公认的三大痛点（对我们有利）

> **痛点A：不要让机器错过偶然性** — 麻生明院士（2025年研讨会）
> 机器按固定程序跑，会忽略意外发现。人机协作可以弥补。

> **痛点B：现有方案缺乏认知与物理多功能性** — Science China Materials 2025
> 学术界承认现有自动化在动态多变场景下不够灵活。

> **痛点C：最耗时的不是在想象新材料，而是在现实中做出来** — MIT Tech Review 2025.12
> 从AI预测材料到实际合成验证之间的鸿沟是目前最大的瓶颈。

## 第三部分：推理出的核心场景

### 3.1 场景选择逻辑

```
在新材料研发中，我们把"纯自动化"和"纯人"各自的局限画出来：

                    做了但不好
                         ↓
    Chemspeed能做的        中间地带（人机协作价值）      人类独有的
    ┌──────────────┐    ┌──────────────────┐    ┌──────────────┐
    │ 称量          │    │ 薄膜涂布的       │    │ 实验设计      │
    │ 液体加样      │    │ 自适应参数优化    │    │ 异常判断      │
    │ 温度控制      │    │                  │    │ 方法创新      │
    │ 标准表征      │    │ 凝胶化点判断      │    │ 机理分析      │
    │ 固定工艺批量  │    │ 浆料流变触感评估   │    │               │
    └──────────────┘    └──────────────────┘    └──────────────┘
    → 标准化、已商用     → 你们的目标区        → 目前只有人能做
```

**为什么薄膜涂布是中间地带的最佳候选？**

因为它是新材料实验室最普遍、最耗时、**对人机协作需求最迫切**的环节。

### 3.2 场景描述：高通量薄膜配方优化

#### 背景

在钙钛矿太阳能电池、有机光电、催化涂层等能源材料研发中，核心工作是**把前驱液涂成薄膜**（旋涂/刮涂/狭缝涂布/喷涂）。

典型流程：
```
配制前驱液（不同配方） → 涂布成膜 → 后处理 → 表征性能
```

#### 现有方案的根本问题

**问题A：工艺-配方耦合**

假设你要高通量跑100个不同配方（不同溶剂比、不同浓度、不同添加剂）。你给每个配方的涂布参数（转速/刀高/速度）应该一样吗？

```
如果全部用同样的参数：
  配方A（粘度刚好）→ 涂布完美 → 膜质量高
  配方B（粘度偏高）→ 涂布很差 → 膜质量低
  结论：配方A比配方B好

但这是真的吗？
如果给配方B调一下涂布参数也能涂好呢？
100个配方的性能差异里，有多少来自配方本身，
有多少来自涂布质量？

这个"工艺-配方耦合"问题，
是薄膜材料研发中一个真实的、但常被忽略的痛点。
```

**问题B：人的触觉经验无法量化**

有经验的实验员拿到一个新配方的前驱液，会"看一眼、摸一下"来评估粘度，然后决定涂布参数。这个"看一眼摸一下"的能力：

- 来自多年经验的积累
- 涉及到触觉判断——溶液在指尖的流动感、在容器壁上的挂壁感
- 无法被现有自动化系统捕捉和使用
- 也无法被有效传承（新实验员需要几年才能培养这个感觉）

**问题C：Chemspeed 做不了自适应**

Chemspeed 可以自动涂100个样品，但每个配方用同样的预设参数。要换参数需要工程师编程介入。它没有"摸一下浆料判断粘度"的能力。

#### 我们的方案价值链

```
Step 1（奠基）：5个配方的"人机教学相长"
  实验员戴触觉手套 → 做5个不同配方的涂布
  每个配方：摸浆料评估 → 凭经验设参数 → 涂布 → 观察膜质量
  系统记录：触觉数据（压力分布、搅拌阻力、挂壁感）
            + IMU轨迹（搅拌动作、涂布动作）
            + 涂布参数
            + 膜质量

Step 2（AI训练）：建立"手感-参数-质量"映射
  AI模型学习：当触觉信号是XXX时，最优涂布参数是YYY
  原理：这不是玄学——触觉信号实际上是浆料流变特性的物理表征

Step 3（高通量）：95个配方自动执行
  新配方浆料进入系统
  → 机械臂夹爪（带触觉传感）评估浆料触感
  → AI推荐最优涂布参数
  → 自动涂布 → 自动后处理 → 自动表征

Step 4（公平对比）：获得真实配方排名
  每个配方用最适合自己的参数跑
  性能差异 ≈ 配方本征性能差异（不是涂布质量差异）
  科学家拿到真实可靠的配方排名
```

#### 为什么Chemspeed做不到？

Chemspeed 的 Rumble 平台是"同参数批量执行"逻辑——要么所有配方同一套参数，要么工程师逐一编程。它缺少的是**自适应能力**，而这个自适应能力需要：
1. 触觉传感来评估浆料状态 → **你们的手套是唯一方案**
2. AI学习"触觉-参数-质量"映射 → **你们可以训练**

#### 为什么纯人手做不到？

人可以每个配方单独调参，但：
- 效率低（一天5-10个）
- 主观差异大（今天手感 ≠ 明天手感）
- 经验无法传承（张博士退休了，他的"手感"就带走了）

|:----|:---------|:-----|
| 触觉传感32×32=1024点 | ✅ 已有 | 够用 |
| 15Hz帧率 | ⚠️ 勉强够 | 涂布是秒级过程，15Hz能记录整体，但抓不到毫秒级事件 |
| 9-DOF IMU | ✅ 已有 | 可追踪手部轨迹 |
| 耐有机溶剂（DMF/NMP/氯苯） | ❓ 待验证 | 防护方案需测试 |
| 可穿戴舒适度 | ⚠️ 需优化 | 长时间做实验的舒适度需要评估 |

#### 关键工程问题清单

1. **传感器溶剂耐受性**：织物传感器能否接触钙钛矿常用溶剂（DMF、DMSO、氯苯、异丙醇等）？如果不能，需要什么防护方案？防护层对灵敏度影响多大？
2. **数据映射**：从"人手触觉数据"到"机械臂夹爪触觉数据"的映射怎么做？人手和夹爪的几何结构完全不同。
3. **标定**：怎么定义"这个触觉信号=这个粘度级别"？需要多少标定数据？
4. **夹爪改造**：现有协作臂夹爪能不能集成你们的触觉传感器？还是需要定制？
5. **系统集成**：手套数据 + IMU + 摄像头 + 协作臂 + AI平台的完整链路怎么打通？

## 第四部分：关于"灵巧手"的结论

调研后的结论：

**你们的方案不依赖灵巧手也能创造价值。**

现有灵巧手（Shadow Hand等）触觉覆盖<20%表面、价格¥100万+、控制复杂。你们的织物传感器覆盖整个手掌，是分布式触觉，比灵巧手更实用。

**建议的技术路线：**
- 用标准协作臂（如UR5 ¥15-20万）+ 你们的触觉夹爪
- 不做灵巧手（技术不成熟、成本高、复杂度大）
- 让触觉传感成为核心差异化，而不是灵巧手本身

## 第五部分：对融资故事的建议

基于本次调研和推理，建议的故事框架：

> **"Chemsped做了25年『已知配方的标准化批量实验』，解决的是『生产端』的效率问题。**
>
> **但我们做新材料研发的人都知道：研发端最大的痛苦不是『已知配方的批量跑』，而是『新配方来了不知道该用什么参数跑』——换一个溶剂比、换一个浓度，原来调好的参数就废了。**
>
> **这个问题在薄膜材料（钙钛矿、有机光电、催化涂层）中尤其突出——所有配方都要涂成薄膜才能测性能，但不同配方的浆料流变性天差地别。**
>
> **有经验的实验员能『摸一下浆料』就知道该用什么参数。但这个『摸一下』的能力没法被传承、没法被规模化。**
>
> **我们做的，就是用触觉手套把这个『摸一下』变成数据，让AI学会『这种手感→这些参数』的映射，然后机器人自动用最优参数跑100个配方。**
>
> **这不是在 Chemspeed 的赛道上和他们竞争，而是在一条他们覆盖不了的赛道上创造新价值。"**

## 第六部分：需要浙大团队讨论的开放问题

### 方向类

1. **场景优先级**：薄膜涂布这个方向符合你们的实际研发需求吗？还是你们手上有更迫切的问题？
2. **Demo选择**：如果要做第一个可行性Demo，建议用钙钛矿还是其他材料体系？
3. **浙大博士的协作**：做钙钛矿的浙大博士是否愿意参与这个方向的验证？

### 技术类

4. **溶剂耐受性**：现有织物手套能不能接触DMF/氯苯等溶剂？谁可以帮忙测试？
5. **实验流程**：能否提供一个你们实验室实际做的薄膜涂布流程（具体到步骤、操作方式、时间尺度）？这对我们做系统设计至关重要。
6. **表征周期**：从涂布完成到拿到性能数据，通常需要多长时间？这决定了闭环迭代的速度。

### 竞争类

7. **你们实验室有没有用过 Chemspeed 或类似自动化设备？** 体验如何？哪些做得好、哪些不好？
8. **你们在配方实验中，是否遇到过"因为涂布参数没调好导致结果不可信"的情况？** 这个频率有多高？
9. **有没有其他场景（除了薄膜涂布）你觉得更适合我们方案？**
